Installing The JDK

Originally by Ashutosh Gupta

Contents

Windows 7

  1. Download and run the appropriate Java Development Kit installer from http://www.oracle.com/technetwork/java/javase/downloads/index-jsp-138363.html#javasejdk

    • To determine if your machine is running 32- or 64-bit,

      • Go to Start

      • Right-click on "Computer"

      • Choose "Properties"

        Under System Type, you should see either "32-bit Operating System" or "64-bit Operating System".

  1. Update your system PATH variable:

    • Go to Start
    • Choose "Control Panel"
    • In the search bar at the top right, type "Path" and press enter
    • Choose "Edit the system environment variables"
    • Click "Environment Variables"
    • In the "System variables" section, scroll to "Path"
    • Select the Path variable and click "Edit"
    • Under "Variable value", add ;C:\Program Files\Java\jdk1.8.0_20\bin

    The system Path variable tells the operating system where to find the Java tools that you installed in step 1. If you did not install the tools in C:\Program Files\Java\jdk1.8.0_20, change that portion of the path to the correct folder. Important: the path must have bin at the end.

    It is important that there is a semicolon (;) separating the folder that you add from the previous folder. For example, if your Path variable looks like this initially:

    C:\Foo\bin;C:\Bar\bin

    it should look like this after you add the new folder:

    C:\Foo\bin;C:\Bar\bin;C:\Program Files\Java\jdk1.8.0_20\bin

    Press "OK" in all prompts until you are back to the Control Panel.

  2. Determine if Java was installed correctly at the command line ( the Windows cmd.exe program):

    • Press the Windows Key + R
    • Type "cmd"
    • Press enter
    • At the command prompt type java -version. You should get output on the next line like java version "1.8.0_20.
    • At the command prompt type javac -version. You should get output like javac 1.8.0_20.

Mac OS X

  1. Download and run the appropriate Java Development Kit installer DMG from http://www.oracle.com/technetwork/java/javase/downloads/index-jsp-138363.html#javasejdk
  2. Double-click on the .dmg file you downloaded to mount it and open it in the Finder.
  3. Double click on the installer .pkg file to run the installer.
  4. Check that the JDK has been properly installed.
    • Run /Applications/Utilities/Terminal.app
    • At the command prompt type java -version. You should get output on the next line like java version "1.8.0_20.
    • At the command prompt type javac -version. You should get output like javac 1.8.0_20.

Linux

  1. Download and run the appropriate Java Development Kit installer from http://www.oracle.com/technetwork/java/javase/downloads/index-jsp-138363.html#javasejdk

OR

For Ubuntu and other debian based distribution, install openjdk using your default package manager

OR

For Fedora-based and other distros that use the Redhat Package Manager format, download the correct "Linux RPM" architecture from Java Downloads
  1. Add the JDK's bin directory to your PATH environment variable, if necessary.
  2. Check that the JDK has been properly installed.
    • Open a terminal window.
    • At the command prompt type java -version. You should get output on the next line like java version "1.8.0_20.
    • At the command prompt type javac -version. You should get output like javac 1.8.0_20.

Troubleshooting

If neither the java nor javac commands work, first try closing the command window and opening a new one so that the command shell reads the PATH environment variable you just updated. If that does not work, double check that the folder you added to the Path variable contains executable files like java and javac on Unix, or java.exe and javac.exe on Windows.

If java works but javac does not, you probably installed a JRE (Java Runtime Environment), not a JDK. Return to Step 1 and start over.
